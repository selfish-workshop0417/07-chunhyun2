# 오늘의 자리 (Today's Seat) — PRD

## 1. 서비스 개요
"오늘의 자리"는 회사 구성원 전체가 서로의 근무 상태(휴가, 조퇴, 재택근무, 해피데이 오전/오후)를 한눈에 확인할 수 있는 사내 근태 현황 대시보드다. 누가 오늘 자리에 있고 누가 없는지 즉시 파악할 수 있어, 커뮤니케이션과 협업의 마찰을 줄이는 것이 목표다.

타겟은 전 직원이며, 2026년 6월 출시를 목표로 한다.

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)

### 3-1. 근무 상태 유형 (8가지)
사용자가 셀프 체크인할 수 있는 근무 상태는 다음과 같다.

| 상태 | 설명 |
|---|---|
| 휴가 | 종일 부재 |
| 반차 | 오전 또는 오후 절반 부재 |
| 병가 | 건강상의 이유로 부재 |
| 조퇴 | 당일 중 일찍 퇴근 |
| 재택근무 | 사무실 외 근무 |
| 해피데이(오전) | 오전 반일 휴무 제도 |
| 해피데이(오후) | 오후 반일 휴무 제도 |
| 출장 | 업무상 외부 일정 (장기) |
| 외근 | 업무상 외부 일정 (당일) |

### 3-2. 상태 입력 방식
- **본인이 직접 입력** (셀프 체크인). 관리자 승인 절차 없음.
- 입력 항목:
  - 날짜 (필수, 단일일 또는 기간)
  - 상태 유형 (필수, 8가지 중 택 1)
  - 시간대 (필수, 종일 / 오전 / 오후 / 특정 시간)
  - 사유 (선택, 자유 텍스트)

### 3-3. MVP 핵심 기능
1. **오늘 부재자 리스트** — 부재자만 표시 (출근자 숨김), 이름·상태·시간대·사유 노출
2. **월간 캘린더 뷰** — 전 직원 통합 캘린더, 날짜 셀 클릭 시 상세 드릴다운
3. **내 상태 등록/수정/삭제** — 본인 상태만 CRUD 가능, 타인은 조회만

### 3-4. 사용자 시나리오
- **A. 휴가 등록** — 접속 → 내 상태 등록 → 날짜·상태·시간대 입력 → 캘린더 반영 확인
- **B. 오늘 누가 없는지 확인** — 접속 → 메인 부재자 리스트 확인
- **C. 월간 일정 확인** — 월간 캘린더 탭 → 날짜 셀 클릭 → 해당일 부재자 상세 확인

### 3-5. 범위 구분
- ✅ **MVP (2026년 6월)**: 상태 8가지 셀프 체크인, 오늘 부재자 리스트, 월간 통합 캘린더, 사내 SSO 로그인
- ⏳ **차기**: Slack/이메일 알림, 휴가 잔여일수 관리, 승인 워크플로우, 부서별 필터, 통계 대시보드, 기존 근태 시스템 연동

### 3-6. 개발 우선순위
1. 상태 등록 UI (데이터 기반)
2. 오늘 부재자 리스트
3. 월간 캘린더 뷰
4. 내 상태 수정/삭제
5. 로그인/계정 연동 (병렬)

상세 문서: `planning/01-features.md`

## 4. 디자인 (Design)
깔끔하고 정보 밀도가 높은 대시보드 비주얼. 한눈에 파악 가능한 색상 코딩(휴가/재택/조퇴/해피데이)이 핵심이다.
화면 흐름의 큰 줄기: 메인(오늘 보드) → 월간 캘린더 → 개인 상세.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
타겟 사용자 페르소나: 팀원의 근무 상태를 확인해야 하는 전 직원.
사용자 여정의 핵심 단계: 접속 → 오늘 보드 확인 → (필요 시) 월간 캘린더 조회.

상세는 `cx/` 폴더 참조. [작성 예정]

사내 서비스이므로 외부 마케팅보다 내부 도입/정착 전략이 핵심.
주요 채널: 사내 공지, Slack/메신저 공유.

상세는 `marketing/` 폴더 참조. [작성 예정]

성장 방향: 근태 → 일정/회의실/좌석 예약 등 사내 운영 통합 허브로 확장 가능.
