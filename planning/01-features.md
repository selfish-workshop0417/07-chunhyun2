# 기능 명세 (Feature Specification)

## 1. 근무 상태 유형 (8가지)

사용자가 셀프 체크인할 수 있는 근무 상태는 다음 8가지로 정의한다.

| 상태 | 설명 | 색상(예시) |
|---|---|---|
| 휴가 | 종일 부재 | 🔴 빨강 |
| 반차 | 오전 또는 오후 절반 부재 | 🟠 주황 |
| 병가 | 건강상의 이유로 부재 | 🟣 보라 |
| 조퇴 | 당일 중 일찍 퇴근 | 🟡 노랑 |
| 재택근무 | 사무실 외 근무 | 🔵 파랑 |
| 해피데이(오전) | 오전 반일 휴무 제도 | 🟢 연두 |
| 해피데이(오후) | 오후 반일 휴무 제도 | 🟢 연두 |
| 출장 | 업무상 외부 일정 (장기) | 🟤 갈색 |
| 외근 | 업무상 외부 일정 (당일) | ⚪ 회색 |

> 추후 색상/아이콘은 디자인 팀에서 확정한다.

## 2. 상태 입력 (셀프 체크인)

### 입력 방식
- **본인이 직접 입력**한다. (관리자 개입 없음)
- 승인 절차 없이 등록과 동시에 반영된다. (MVP 기준)

### 입력 항목
| 필드 | 필수 여부 | 설명 |
|---|---|---|
| 날짜 | 필수 | 단일 날짜 또는 기간(시작일~종료일) |
| 상태 유형 | 필수 | 위 8가지 중 택 1 |
| 시간대 | 필수 | 종일 / 오전 / 오후 / 특정 시간(HH:MM ~ HH:MM) |
| 사유 | 선택 | 자유 텍스트 (예: "가족 행사") |

## 3. MVP 핵심 기능

### 3-1. 오늘 부재자 리스트
- 첫 진입 화면(메인).
- **부재자만** 리스트로 표시 (사무실 근무자는 숨김).
- 각 항목에 표시할 정보:
  - 이름
  - 상태 유형 (색상/아이콘)
  - 시간대 (예: "종일", "오전만", "14:00 이후")
  - 사유 (입력된 경우)
- 정렬: 부서 → 이름 순 (기본)

### 3-2. 월간 캘린더 뷰
- **전 직원 통합 캘린더**.
- 한 화면에 전체 월이 보이고, 각 날짜 칸에 해당 일의 부재자가 요약 표시된다.
- 날짜 셀 클릭 시 해당 날짜의 상세 부재자 리스트로 드릴다운.
- 월 이동 (이전/다음) 가능.

### 3-3. 내 상태 등록/수정/삭제
- 로그인한 사용자 본인의 상태만 CRUD 가능.
- 타인의 상태는 **조회만** 가능.

## 4. 사용자 시나리오 (MVP)

### 시나리오 A: 휴가 등록
> 김직원은 다음 주 월요일에 연차를 쓴다.
1. "오늘의 자리"에 접속한다.
2. "내 상태 등록" 버튼을 누른다.
3. 날짜: 2026-04-20, 상태: 휴가, 시간대: 종일, 사유: 비움 → 저장.
4. 월간 캘린더 2026-04-20 칸에 본인 이름이 추가됨을 확인한다.

### 시나리오 B: 오늘 누가 없는지 확인
> 박팀장은 아침에 팀원 상태를 파악하려 한다.
1. "오늘의 자리"에 접속한다.
2. 메인 화면의 부재자 리스트를 본다.
3. "오늘은 3명 부재 (이○○ 휴가, 최○○ 재택, 정○○ 외근)" 확인.

### 시나리오 C: 월간 일정 확인
> 최팀원은 이번 달 팀 휴가 분포를 확인하려 한다.
1. 메인 → 월간 캘린더 탭 클릭.
2. 4월 달력이 펼쳐짐.
3. 특정 날짜 셀 클릭 → 그 날의 부재자 상세 리스트 확인.

## 5. 범위 구분

### ✅ MVP (1차 출시, 2026년 6월 목표)
- 근무 상태 8가지 셀프 체크인 (등록/수정/삭제)
- 오늘 부재자 리스트
- 월간 전 직원 통합 캘린더
- 로그인/본인 식별 (사내 SSO 활용 가정)

### ⏳ 차기 (MVP 이후)
- Slack/이메일 알림
- 휴가 잔여일수 관리
- 승인 워크플로우 (관리자 승인)
- 부서별 필터 및 그룹 뷰
- 통계 대시보드 (월별/부서별 부재율)
- 기존 근태 시스템(플렉스 등) 연동

## 6. 우선순위

| 순서 | 기능 | 이유 |
|---|---|---|
| 1 | 상태 등록 UI | 데이터가 있어야 나머지가 동작 |
| 2 | 오늘 부재자 리스트 | 가장 자주 쓰는 메인 뷰 |
| 3 | 월간 캘린더 뷰 | 두 번째 핵심 뷰 |
| 4 | 내 상태 수정/삭제 | 등록 이후 필연적 |
| 5 | 로그인/계정 연동 | 인프라 요소 (병렬 작업 가능) |
