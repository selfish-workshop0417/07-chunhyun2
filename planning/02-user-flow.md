# 화면 플로우 (User Flow)

"오늘의 자리" 서비스의 주요 화면 흐름을 정의한다. MVP 범위 내 3개 주 사용자 시나리오에 대응한다.

## 1. 전체 화면 맵

```
[로그인]
   │
   ▼
[메인 = 오늘 부재자 리스트] ──────┐
   │                             │
   ├─▶ [내 상태 등록/수정 모달]    │
   │                             │
   └─▶ [월간 캘린더 뷰] ◀─────────┘
           │
           ▼
      [일자별 상세 부재자 리스트]
```

## 2. 주요 화면 설명

### 2-1. 로그인 화면
- 사내 SSO 버튼 (단일 진입점)
- 성공 시 → 메인으로 이동

### 2-2. 메인 (오늘 부재자 리스트)
- 상단: 오늘 날짜, "오늘은 N명 부재" 요약
- 중단: 부재자 카드 리스트
  - 이름 / 상태 아이콘·색상 / 시간대 / 사유 (있을 때)
- 상단 우측: "내 상태 등록" 버튼
- 하단/좌측: "월간 캘린더" 탭 전환 버튼

### 2-3. 내 상태 등록/수정 모달
- 진입: 메인 "내 상태 등록" 또는 본인 카드 클릭
- 입력 순서:
  1. 날짜 선택 (단일일 또는 기간)
  2. 상태 유형 선택 (8가지 중 택 1, 칩 형태 버튼)
  3. 시간대 선택 (종일/오전/오후/특정 시간)
  4. 사유 입력 (선택)
- 하단: "저장" / "삭제"(수정 시) / "취소" 버튼

### 2-4. 월간 캘린더 뷰
- 상단: 월 네비게이션 (← 이전달 / 2026.04 / 다음달 →)
- 본체: 7×5~6 그리드. 각 셀에 그날 부재자 이름 최대 3명 + "외 N명" 요약
- 셀 클릭 → 일자별 상세로 전환
- 우상단: "메인으로" 버튼

### 2-5. 일자별 상세 부재자 리스트
- 해당 일자의 전체 부재자 리스트 (메인과 동일 카드 포맷)
- 상단: 날짜, "← 캘린더로 돌아가기"

## 3. 시나리오별 플로우

### 시나리오 A: 휴가 등록
```
로그인
  → 메인
  → [내 상태 등록] 클릭
  → 모달에서 날짜·상태·시간대 입력
  → [저장]
  → 메인으로 돌아옴 (본인 상태가 부재자 리스트에 반영됨)
```

### 시나리오 B: 오늘 부재자 확인
```
로그인
  → 메인
  → 부재자 리스트 즉시 확인 (추가 클릭 불필요)
```

### 시나리오 C: 월간 일정 확인
```
로그인
  → 메인
  → [월간 캘린더] 탭 클릭
  → 원하는 날짜 셀 클릭
  → 일자별 상세 리스트 확인
  → [← 캘린더로 돌아가기]로 복귀
```

## 4. 진입 깊이 원칙

| 시나리오 | 로그인 이후 클릭 수 |
|---|---|
| 오늘 부재자 확인 | 0 클릭 (즉시 표시) |
| 월간 조회 | 1~2 클릭 |
| 내 상태 등록 | 1~4 클릭 (모달 내부 포함) |

→ **"가장 자주 쓰는 동작(오늘 확인)을 가장 얕은 깊이에 둔다"**가 원칙.

## 5. 예외/엣지 케이스

- **오늘 부재자가 없을 때**: 메인에 "오늘은 전원 사무실 근무입니다" 상태 메시지 표시
- **과거 날짜에 상태 등록 시도**: 가능 (사후 기록용), 단 오늘 이전은 "기록" 라벨로 구분
- **동일 날짜에 상태 중복 등록 시도**: 기존 상태를 덮어쓸지 확인 모달
- **기간 등록 중 일부 날짜만 이미 다른 상태**: 덮어쓸 대상 날짜를 미리 보여주고 확인
